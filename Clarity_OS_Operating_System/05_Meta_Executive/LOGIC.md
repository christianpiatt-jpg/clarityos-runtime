# Meta Executive Logic 
Meta Executive Logic defines how the system evaluates decisions, patterns, and operator state. 
It ensures the operator is not acting from distortion, drift, or external hijack. 
 
## Core evaluation questions 
- Is the operator inside their own frame? 
- Is the ULDA loop being followed or skipped? 
- Is the decision proportional to the situation? 
- Is pressure distorting perception? 
- Is the environment stable enough for clean action? 
 
## Meta-level invariants 
- No decision is valid if Diagnose was skipped. 
- No command is valid if the operator's frame is collapsed. 
- No adjustment is valid if it violates geometry constraints. 
 
## Failure modes 
- Overclocking: acting too fast for ULDA to run. 
- Collapse: losing agency and reacting instead of deciding. 
- Drift: slowly leaving the intended frame without noticing. 
- Hijack: someone else's frame replacing the operator's. 
 
## Outputs 
- Corrections to the Executive Layer. 
- Requests for ULDA re-run. 
- Boundary adjustments. 
- Stability recommendations.
