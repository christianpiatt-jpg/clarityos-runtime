# Environmental Geometry 
Environmental Geometry defines the space, frames, and fields in which all OS activity occurs. 
It is the substrate that ULDA, Executive Layers, and Propagation all operate inside. 
 
## Purpose 
- Provide a stable spatial model for interpreting signals. 
- Define frames, boundaries, and fields that shape behavior. 
- Anchor ULDA phases in a real environment. 
 
## Relationship to the OS 
- ULDA uses geometry to Locate and Diagnose. 
- Martial OS uses geometry to understand pressure and conflict. 
- Executive Layer uses geometry to issue commands into a space. 
- Propagation OS uses geometry to model spread across fields. 
 
## Files in this subsystem 
- PRIMITIVES.md: the basic building blocks of geometry. 
- RULES.md: how geometry behaves and constrains systems.
