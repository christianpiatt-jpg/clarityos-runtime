# Geometry Rules 
Geometry rules define how environments behave and how systems move inside them. 
These rules ensure that the OS behaves predictably across contexts. 
 
## Core rules 
- No signal exists without a location. 
- No action is context-free; all actions occur inside a frame. 
- Boundaries shape behavior more than intentions. 
- Fields influence movement even when invisible. 
- Regions define meaning; crossing regions changes interpretation. 
 
## Dynamic rules 
- Pressure accumulates along boundaries. 
- Flow follows channels unless blocked. 
- Systems drift toward lower resistance. 
 
## Why rules matter 
ULDA uses rules to Diagnose patterns. 
Executive Layer uses rules to issue effective commands. 
Propagation OS uses rules to predict spread.
