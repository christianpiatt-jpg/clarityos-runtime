# Executive Commands 
Commands are the atomic units of action in the OS. 
They must be clear, bounded, and tied to a diagnosed need. 
 
## Command types 
1. Boundary Command 
   - Establish, adjust, or reinforce a boundary. 
 
2. Inquiry Command 
   - Request information needed for ULDA. 
 
3. Adjustment Command 
   - Make a targeted change to stabilize a system. 
 
4. Commitment Command 
   - Declare a decision and its follow-through. 
 
5. Delegation Command 
   - Assign responsibility with clarity and boundaries. 
 
## Command grammar 
A command must include: 
- What is being done. 
- Why it is being done (ULDA Diagnose). 
- When it will occur. 
- What boundary or constraint applies. 
 
## Command integrity 
A command is only complete when: 
- It is issued. 
- It is acknowledged. 
- It is executed. 
- It is reviewed.
