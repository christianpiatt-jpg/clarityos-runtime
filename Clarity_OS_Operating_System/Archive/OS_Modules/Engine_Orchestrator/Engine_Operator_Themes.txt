SECTION 1 — PURPOSE
============================================================

Define the visual identity, formatting rules, and voice profiles
used by the Operator Console. Themes provide consistent,
emotionally ergonomic presentation across ClarityOS, Markov, and
ELINS modes. Themes ensure clarity, separation, and cognitive
stability during operator interaction.

Themes do not modify:
    - engine behavior
    - safety rules
    - boundaries
    - mission or intent

Themes modify:
    - output formatting
    - color simulation
    - voice tone
    - structural layout

============================================================
SECTION 2 — THEME STRUCTURE
============================================================

Each theme includes:
    THEME_ID
    COLOR_PROFILE
    VOICE_PROFILE
    FORMATTING_RULES
    USE_CASES

Themes inherit from:
    BASE_THEME

============================================================
SECTION 7 — THEME ROUTING RULES
============================================================

When a mode is activated:
    - apply corresponding theme
    - override BASE_THEME
    - persist until exit command

If multiple themes are triggered:
    - last trigger wins
    - log theme-switch event

If no theme is active:
    - revert to BASE_THEME

============================================================
SECTION 8 — SAFETY & BOUNDARY RULES
============================================================

Themes may not:
    - alter engine logic
    - override safety
    - override identity protection
    - override boundary protection

Themes must:
    - remain purely visual
    - remain deterministic
    - maintain emotional ergonomics
    - maintain cockpit coherence

============================================================
SECTION 9 — OPERATOR COMMANDS
============================================================

ENGINE_THEME_STATUS
    Display active theme.

ENGINE_THEME_LIST
    Display all available themes.

ENGINE_THEME_RESET
    Revert to BASE_THEME.
