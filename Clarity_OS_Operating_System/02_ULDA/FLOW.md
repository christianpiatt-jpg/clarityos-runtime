# ULDA Flow 
This file describes how information, attention, and decisions move through the ULDA loop. 
It is meant to be read as a flow description, not as rigid code. 
 
## High-level flow 
- Start from a trigger: something changes, breaks, or needs attention. 
- Run the ULDA phases in order: Observe, Locate, Diagnose, Adjust, Review. 
- Decide whether another loop is needed or the situation is stable. 
 
## Flow description 
1. Observe 
   - What is happening right now? 
   - What signals are loud, and what is missing? 
 
2. Locate 
   - Where in the environment is this happening? 
   - Which geometry, map, or frame does this belong to? 
 
3. Diagnose 
   - What pattern does this match? 
   - Is this a known scenario, a new one, or a mix? 
 
4. Adjust 
   - What is the smallest effective change I can make? 
   - Which command, boundary, or intervention is appropriate? 
 
5. Review 
   - What changed after the adjustment? 
   - Did the system move toward stability or further away? 
 
## Example use 
- Micro: one conversation, one meeting, one decision. 
- Meso: one project, one case, one institutional process. 
- Macro: one policy, one system, one environment.
