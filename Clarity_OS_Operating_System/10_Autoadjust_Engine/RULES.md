# Autoadjust Rules 
The Autoadjust Engine uses rule-based detection to identify operator state and system drift. 
 
## Drift Detection Rules 
- If Diagnose is skipped, trigger ULDA re-run. 
- If operator language shows overwhelm, activate Stability Buddy. 
- If boundaries weaken, activate Martial OS. 
 
## Pressure Detection Rules 
- If pressure spikes, slow the loop. 
- If pressure distorts perception, trigger Meta-Executive audit. 
 
## Altitude Detection Rules 
- If operator speaks in system terms, shift to Founder Console. 
- If operator speaks in basin/field terms, shift to Global Console. 
- If operator speaks in planetary terms, shift to Planetary Console. 
- If operator speaks in moment-to-moment terms, shift to Console v2. 
 
## Buddy Activation Rules 
- Drift Buddy for confusion or misalignment. 
- Stability Buddy for pressure or overload. 
- Clarity Buddy for interpretive fog. 
- Decision Buddy for command formation. 
- Review Buddy for post-action reflection.
