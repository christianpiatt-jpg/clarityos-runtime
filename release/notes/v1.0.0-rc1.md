## 1. Summary — Phases 1 → 12

The engine cohort is a deterministic, read-only operator-intelligence stack built
on top of the existing telemetry/continuity layers. Every reasoning engine is a
pure function: no wall-clock, no randomness, no ML, no inference, no
`operator_state` writes.

## 2. Major capabilities

- **Temporal forecast** — drift velocity/acceleration, coherence trend, stability
  forecast, and a four-state trajectory classification.
- **Causal reasoning** — first-order graph, multi-hop influence/centrality,
  structural motifs (loops/bottlenecks/attractors), ranked multi-chains, deltas,
  stability forecast, and unified temporal-causal narrative.
- **Behavioral intelligence** — operator actions as first-class causal atoms;
  action loops / trigger chains / habits / bottlenecks / attractors.
- **Behavioral forecasting** — next-action prediction, habit trajectory, trigger
  likelihood, loop continuation; behavioral deltas, stability, and narrative.
- **Recommendations** — six deterministic structural reason types
  (habit_weakening / trigger_volatility / loop_break / bottleneck_relief /
  attractor_alignment / forecast_alignment) + explanation narrative.
- **Tri-surface surfacing** — read-only operator-console tiles on web, desktop,
  and phone.

## 3. Deterministic guarantees

- Every engine function is pure: **no** I/O, wall-clock, randomness, ML, or
  inference; identical inputs yield byte-identical, JSON-serialisable output.
- Every output collection is fully sorted with explicit tiebreakers — **no**
  nondeterministic iteration.
- Timestamps are caller-supplied; the only component permitted the wall-clock is
  `operator_state` (metadata-only persistence), which the engines never write.
- Production dependencies are exact-pinned (`requirements.txt`), enforced by
  `tests/test_repo_hygiene.py::TestDependencyFreeze`; dev-only tools are isolated
  in `requirements-dev.txt`.

## 5. Test + performance certification

- **Backend:** `pytest tests/` — **8768 passed, 0 failed** (12.0 sweep, re-confirmed
  after the 12.2 performance pass — identical pass count proves the optimizations
  are behavior-preserving).
- **Web:** vitest — 639 passed (90 files). **Desktop:** vitest — 17 passed.
  **Phone:** type-clean (`tsc --noEmit`), structurally complete (no harness).
- **Performance (12.2):** `compute_node_centrality` and the behavioral/causal
  degree scans reduced from O(N·E) to O(N+E) (float-order / integer-exact —
  zero behavior change), benefiting the action-augmented behavioral graph as the
  action log grows. No debug logs, no dead paths.

## 6. Known issues / release-deferred

1. **Surfacing wiring (deferred by design).** The `behavioral_forecast` (10.x) and
   `recommendation_narrative` (11.x) tiles are implemented and tested against the
   exact engine output shapes, but `GET /operator/telemetry` does **not yet emit**
   those two keys (10.4 / 11.2 were UI-only by mandate). Those tiles collapse to
   their heading until a small backend-wiring step adds the two keys. The
   `behavioral_motifs` key (9.5) is emitted live.
2. **Tag is annotated, not GPG-signed.** GPG signing is not configured in this
   environment; RC1 ships as an annotated tag. Configure a signing key and re-tag
   (or sign RC2/GA).
3. **Working-tree provenance.** The branch carries accumulated multi-session work
   beyond the engine cohort (deploy artifacts, unrelated modules, product-side
   edits); the RC commit is scoped to the engine-cohort sources — see the tag
   message for the exact included set.
