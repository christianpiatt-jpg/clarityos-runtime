## Runtime architecture

The runtime is partitioned into five boundaries plus one cross-cutting
helper module. Eager imports flow strictly downward (BD1 → BD5); the
three lazy back-edges are documented in
[`docs/boundaries.md`](../boundaries.md).

| BD  | Module                    | Responsibility                                                                                  |
| --- | ------------------------- | ----------------------------------------------------------------------------------------------- |
| BD1 | `app.py`                  | HTTP routes, auth, CORS, error envelope, rate limit                                             |
| BD2 | `intelligence_kernel.py`  | ESO funnel, model funnel, audit funnel, S_ELINS QC                                              |
| BD3 | `model_router.py`         | Provider selection, outbound HTTP, mock fallback, vault-backed founder default                  |
| BD4 | `operator_state.py`       | Per-user preferences, history, decay; sole storage dep is BD5                                   |
| BD5 | `memory_vault.py`         | Per-user encrypted KV (PBKDF2 + HMAC-CTR + HMAC-SHA256 MAC); namespace allow-list                |
| —   | `runtime_privacy.py`      | Pure-string redaction helpers (`user_ref`, `session_ref`, `prompt_preview`, `topic_trim`, `event_ref`) |

## Locked invariants

39 invariants across the five boundaries plus the cross-cutting
facade. Each is backed by a direct assertion in
`tests/test_runtime_inv_*.py` plus at least one stability test in the
PASS-5 / PASS-6 runtime suites. The full table lives in
[`docs/invariants.md`](../invariants.md). Headline contracts:

### BD3 — Router

* **INV-R1** `select_model` precedence: `override > founder default > preferred_model > task default`. `"auto"` is a sentinel that falls through.
* **INV-R2** Founder default is vault-backed under `__founder_global__ / founder_global.default_model`.
* **INV-R3** Provider HTTP timeout is per-context via `_PROVIDER_HTTP_TIMEOUT_VAR: ContextVar`.
* **INV-R4** `_mock_result` preview uses `runtime_privacy.prompt_preview` (60-char cap, no ellipsis).

### BD2 — Kernel

* **INV-K1** `_macro_seq_lock` is pre-allocated at module import — no lazy-init race.
* **INV-K2** Macro `run_id` format is `macro_<ts_ms>_<seq>` with strictly monotonic seq.
* **INV-K3** Every `run_*` path surfaces `model_id` in its result and in `kernel_logging.meta`.

### BD4 — State

* **INV-S1** `_next_seq` is strictly monotonic per prefix under `_SEQ_LOCK`.
* **INV-S2** `HISTORY_MAX = 200` enforced on live and migration paths.
* **INV-S3** `_strip_forbidden` removes exactly `{"text", "scenario_text", "input_text", "raw_text"}`.

### BD5 — Vault

* **INV-V1** `_KEY_CACHE` entries are `(key_bytes, created_at)` tuples; TTL = 3600 s.
* **INV-V2** `_derive_key` returns deterministic PBKDF2 output.
* **INV-V4** `CLARITYOS_VAULT_PLAINTEXT` enables only on explicit `"true"`.
* **INV-V5** Plaintext warning fires exactly once per process.
* **INV-V6** `ALLOWED_NAMESPACES` is a frozen 12-entry registry.
* **INV-V8** `_secret()` raises `RuntimeError` on missing `CLARITYOS_VAULT_SECRET` — no default key.

### BD1 — HTTP

* **INV-H1 / H2** No raw `user_id` / `session_id` in any FIX-P5-scoped logger.
* **INV-H3** `/billing/intent/confirm` response is field-projected — no `client_secret`, no metadata.
* **INV-H4** `/me/billing` distinguishes `billing_state == "failed"` from `"none"`.

## PASS-5 — Stabilization

Six concurrency + multi-instance test surfaces, mapped one-to-one onto
the PASS-4 fixes:

* **D1** Multi-instance founder-default consistency.
* **D2** `_PROVIDER_HTTP_TIMEOUT_VAR` per-context isolation under
  50 concurrent threads, each in its own `_request_timeout` block.
* **D3** `_next_seq` returns a contiguous unique set under 50-thread
  contention; distinct prefixes are independent.
* **D4** 50 concurrent `run_macro_ELINS` invocations produce 50 unique
  `macro_<ts_ms>_<seq>` ids.
* **D5** `_derive_key` returns deterministic PBKDF2 bytes under 50-thread
  cache-TTL races; cache ends with a single coherent entry per user.
* **D6** 20 concurrent HTTP clients each run login → /me → /elins/preview
  → /me with zero raw user/session leaks across the five FIX-P5
  loggers, zero 500s, and per-user `last_model_used` populated.

## Known limitations

* `.github/workflows/deploy.yml` does NOT build or push a real
  container image. PASS-7 activates that.
* Real provider HTTP calls (OpenAI, Anthropic, Gemini) require env
  keys at runtime; without them the deterministic mock contract
  applies (by design — see BD3 in `docs/boundaries.md`).
* Firestore-backed integration tests are not run in CI; the mock
  backend covers every spine invariant. Production deploys exercise
  Firestore directly.
