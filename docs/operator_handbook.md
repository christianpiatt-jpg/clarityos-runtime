# Operator Handbook

## Purpose

The Operator Handbook is the entry document for operators. It orders the
ClarityOS subsystem documentation into a single reading sequence.

## Structure

1. **Introduction** — purpose and scope of ClarityOS.
2. **Geometry System** — visual and structural primitives. See
   `docs/geometry_system.md`.
3. **Cognitive Pipeline** — the five-layer reasoning sequence. See
   `docs/cognitive_pipeline.md`.
4. **Cockpit** — the operator workspace. See `docs/cockpit/cockpit_spec.md`.
5. **Vault** — persistent memory and structure. See
   `docs/memory_vault.md` and `docs/vault_store.md`.
6. **ELINS** — the report and intelligence subsystem. See
   `docs/elins/elins_deep_spec.md` (core pipeline and regression surfaces)
   and `docs/elins/elins_report_visual.md` (output surfaces).
7. **Runtime** — the process and diagnostics surface. See
   `docs/runtime_ui.md`.

## Reading Order

The handbook is read in section order. Sections 2 through 7 specify subsystems.
General rules and vocabulary are documented within those subsystem docs and in
the methodology/meta docs.
